명세서 · 사진 업데이트 런처
================================================
주기(매주 등)로 신규 이지폼 명세서 + 톡 작업지시서 사진을 받아 정리하는 도구.
순서대로 1 → 2 → 3 → 4 만 돌리면 '데이터 수집' 끝.
그 다음 '웹 반영'(비전추출·매칭·업로드)은 클로드한테 알려주면 진행.


────────────────  1단계: 데이터 수집 (런처 4개)  ────────────────

[ 사진 ]
 1_사진_Chrome디버그열기.bat
    - 디버그 Chrome 이 열림. (처음 1회만) 카톡 로그인 → 이지폼방 '사진모음' 열기.
    - ★ 사진모음을 '최신순'(최신이 위) 으로 정렬해두세요.

 2_사진_캡처다운로드.bat
    - 신규 사진 자동 다운로드 + 날짜이름 정리 + 중복 정리까지 한 번에.
    - 'selected N' 로그의 N 이 작으면 정상(신규만).

[ 명세서 ]   ※ 먼저: 이지폼에서 26년 매출거래목록 CSV 2개를 '새로' 내보내기

   ┌─ CSV 넣는 곳 (★여기에만 덮어쓰기★) ───────────────────────────┐
   │  폴더 :  C:\Users\USER\Desktop\hdsign\easyform-data\           │
   │  파일 :  26년도매출거래목록(주식회사).csv                       │
   │          26년도매출거래목록(개인).csv                          │
   │  (이지폼 [엑셀(csv)] 내보내기 → 위 이름으로 이 폴더에 저장/덮어쓰기) │
   └──────────────────────────────────────────────────────────────┘

   !!! 절대 넣으면 안 되는 곳 !!!

         CSV 를 여기 넣거나 .json 을 덮어쓰면 데이터가 깨집니다. 손대지 마세요.

 3_명세서_주식회사.bat
    - 이지폼: 매출거래목록 '주식회사' / 역순보기 끄기(정순) / 창 최대화 / 1번째 행 클릭.
    - 진행: CSV점검 → [키] → 자동 스크롤 → '마지막 추출 5건' 표시
            → 이지폼에서 그 '다음' 명세서 상세를 직접 열기 → [키]
            → ★상세명세서에 포커스 주고 가만히★ → 5초 뒤 추출 → 보강까지 자동.

 4_명세서_개인.bat
    - 이지폼을 '개인' 목록으로 바꾸고(정순/최대화/1번째 행) 위와 동일.


────────────────  2단계: 웹 반영 (클로드)  ────────────────

 위 1~4 끝나면 클로드한테 "데이터 수집 끝났어" 라고 말하기.
 → 비전추출 → 매칭(matches.json) → R2 업로드(사진/명세서/priced_index) → 백엔드 재시작.
 (비전은 클로드(소넷)가 필요해서 .bat 으로 못 만듭니다.)


────────────────  참고 / 문제 해결  ────────────────

 - 매크로 도는 동안 마우스/키보드 만지지 마세요. 즉시 중단 = Ctrl+Esc 를 '꾹'.
 - 추출이 안 돌면: 포커스가 'cmd 창'이 아니라 '상세명세서'에 있어야 함.
 - 사진은 일반 Chrome 아니라 '1번'으로 띄운 디버그 Chrome 이어야 함.
 - _기타(진단,참고)\ : 평소 안 쓰는 도구(명세서 시작점 확인 / 추출 시험).
 - 비전_매칭_규칙.md : '비전 돌려줘' 할 때 적용되는 추출·매칭 규칙(사장님이 정한 것).
 - 런북_전체절차.md : 상세 절차·원리.
 - ⚠ R2 키가 노출됐으면 Cloudflare 에서 교체 + Railway 환경변수 갱신.
