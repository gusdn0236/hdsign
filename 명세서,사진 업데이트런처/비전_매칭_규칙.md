# 비전 추출 · 매칭 규칙 (사장님이 정한 규칙)

> "비전 돌려줘" 할 때 이 규칙대로 동작합니다. 구현 위치도 함께 적어둠.

## 1. 사진 비전 추출  — 소넷(Sonnet)으로, **구독요금(API 비용 0)**
각 작업지시서 사진에서 아래를 읽어 기록:
- **거래처** : 사진 **상단 표**(거래처명). 상단을 중점으로 봄.
- **발주일 / 출고일(납기일)** : 상단 표. **앞 날짜=발주, 뒤 날짜=출고**. (월-일만 — 사진엔 연도 없음)
- **현장명** : 상단 표.
- **자재 / 품목** : 본문의 재질·가공 텍스트 (갈바·아크릴·스텐·포맥스·잔넬·스카시·고무스카시·골드스텐·
  후렘·레이저타공·측광·후광·전후광·도장·시트컷팅·LED 등). **간판에 들어갈 브랜드/상호 글자는 제외.**
- **사이즈** : 규격 (H:124, 1200*400, 30mm 등).
- 한 사진에 지시서가 여러 개면 다 기록.
- 구현: `auto-quote-data/learning/vision_new_photos.js` (신규), `resume_vision_extract.js` 의 프롬프트.
- 산출: 청크 JSON → `photo_extractions.json` 으로 병합.

## 2. 연도 확정 (비전이 연도를 못 읽으므로)
사진엔 월-일만 있어서, **파일명 날짜(=업로드 시각) 근방의 명세서 연도로 확정**.
12월/1월 연말경계만 ±1년 교정. (재발 거래처를 다른 해 명세서에 잘못 붙이지 않게)

## 3. 사진 ↔ 명세서 매칭 (N:N)
- **거래처 일치** : 괄호(과천/담당자명)·법인표기((주)/주식회사) 제거 후 같으면 일치. (포함 관계도 허용)
  - 예) `(주)진성커뮤니티` = `진성커뮤니티(과천)` = `진성커뮤니티` → 모두 같은 거래처로 봄.
  - 비전 오타(`진성커뮤티티` 등)는 글자가 달라 매칭 안 됨(일부러 — 다른 회사 오합침 방지).
- **날짜 창** : 명세서 날짜가 **[발주일 ~ 출고일 + 7일]** 안에 들어야 함.
- **자재/품목/규격 겹침 점수** (강한 신호일수록 우선):
  1) 규격(사이즈) 숫자 일치 (가장 강함)
  2) 품목코드 정확 일치
  3) 자재 키워드 겹침 + 단일치수(H:xx) 일치
  - 흔한 키워드만 살짝 겹치는 건 수용 안 함(같은 창+거래처라도 과다연결 방지).
- **N:N** : 작업지시서 1장 ↔ 명세서 여러 개(쪼개 끊은 경우), 명세서 1건 ↔ 사진 여러 장 모두 허용.
- 구현: `auto-quote-data/learning/match_photos_to_invoices_v4.py`
- **정본 저장**: `match_photos_to_invoices_v4.py` → `build_matches.py` → **`matches.json`**
  (사진↔명세서 + 사진파일명·날짜·자재·규격·점수·방법. 매 주기 누적.)

## 4. 매칭 데이터가 쓰이는 곳
- **단가찾아보기** : 명세서(가격/규격/grid) + **매칭된 과거 지시서 사진(갤러리)**
- **매출분석** : 명세서(매출 집계)만. 사진은 안 띄우지만, matches.json 에 **사진 파일명이 연결**돼 있어 추후 연동 가능.

## 5. 못 맞추는 경우 (정상)
- 06-09 작업 사진인데 **명세서가 06-10+ 발행** → 아직 명세서 없음 → **다음 주기에 자동 매칭**.
- 비전이 거래처명을 오독한 경우.
- 사진이 참고용이거나 청구 안 된 작업.
